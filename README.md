# analyse-ventes-ecommerce_olist
Analyse multidimensionnelle (Ventes, Logistique et Marketing) d'une plateforme E-commerce avec SQL et Python

# 🛒 Analyse de la Performance E-commerce (Olist)

## 📌 Objectif du Projet
Exploiter une base de données relationnelle complexe pour extraire des indicateurs de performance (KPIs) et optimiser la stratégie marketing d'une marketplace.

## 🛠️ Stack Technique
- **Traitement de données :** [Pandas](https://pandas.pydata.org) (Jointures complexes, agrégations).
- **Analyse Temporelle :** Étude de la saisonnalité des ventes.
- **Marketing :** Segmentation client via la méthode **RFM** (Récence, Fréquence, Montant).

## 🚀 Ce que contient ce projet
- [ ] Analyse du Chiffre d'Affaires par catégorie (Top 5).
- [ ] Traduction automatisée des catégories (Portugais -> Anglais).
- [ ] Visualisation de la croissance mensuelle.

📈 Recommandations Stratégiques (Insights Business)
Basé sur la segmentation RFM et l'analyse de performance, voici les leviers d'optimisation identifiés pour la plateforme :
1. Rétention des "At Risk" (Priorité Haute)
"Le segment 'At Risk' représente [X]% de la base client. Ce sont d'anciens clients fidèles qui n'ont pas commandé depuis plus de 6 mois.
Action : Lancer une campagne de réactivation avec un coupon de réduction personnalisé sur leur catégorie de produit préférée (identifiée via le Top 5 des ventes)."
2. Capitaliser sur les "Champions"
"Nos meilleurs clients génèrent [Y]% du Chiffre d'Affaires total malgré leur faible nombre.
Action : Créer un programme VIP 'Early Access' pour tester les nouveaux produits en avant-première et sécuriser leur fidélité à long terme."
3. Optimisation du Panier Moyen (AOV)
"L'analyse montre une stagnation du panier moyen lors des pics de vente (Black Friday).
Action : Implémenter des algorithmes de Cross-selling (produits complémentaires) au moment du checkout pour augmenter la valeur de chaque commande unique."
